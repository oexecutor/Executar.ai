# ADR-002 — Tokens independentes da plataforma
- **context**: pacote Swiss SRC-04 (`TOK-DESKGO-SWISS-001`) já define tokens aprovados; reimplementar geraria divergência (RISK-005).
- **problem**: como consumir tokens sem duplicar valores?
- **decision_drivers**: fonte única (BR-001); não duplicar (RG-25); valores herdados PROVISIONAL (SEC-003).
- **options_considered**: (a) copiar valores para o pacote; (b) referenciar por ID à fonte canônica externa.
- **decision**: a fonte canônica é **estruturada e externa** (`TOK-DESKGO-SWISS-001`); AMES **referencia por ID** e nunca copia. `tokens.css` é saída gerada.
- **positive_consequences**: sem divergência; portabilidade DTCG; hard-coding proibido (TK-002).
- **negative_consequences**: dependência de disponibilidade do pacote de tokens.
- **risks**: LAC-COLOR/LAC-FONT permanecem até validação na origem.
- **validation**: nenhum valor visual hard-coded em componente (TEST-TOKEN-01).
- **requirement_refs**: [NFR-004, SEC-003, SWISS-006]
- **test_refs**: [TEST-TOKEN-01]
