# ADR-001 — Conteúdo estruturado como fonte canônica
- **context**: SRC-01/SRC-02 mostram que composição visual não substitui dados estruturados; agentes precisariam de OCR (EDS-031).
- **problem**: onde vive a verdade do conteúdo?
- **decision_drivers**: legibilidade por agente (NFR-001); rastreabilidade (BR-003); uma fonte por formato (BR-001).
- **options_considered**: (a) HTML visual como fonte; (b) modelo semântico estruturado (JSON/YAML/CSV) como fonte.
- **decision**: o **modelo de conteúdo semântico** (`content-model.schema.json`) é a fonte canônica. Toda saída visual deriva dele.
- **positive_consequences**: leitura por agente sem OCR; par visual+estruturado (ADR-007); rastreabilidade.
- **negative_consequences**: exige normalização (G1) antes de gerar.
- **risks**: custo de modelagem inicial.
- **validation**: exemplos validam contra o schema (ACR-011).
- **requirement_refs**: [FR-001, DR-001, NFR-001, AI-001]
- **test_refs**: [TEST-SCHEMA-01, TEST-AI-01]
